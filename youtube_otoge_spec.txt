YouTube音ゲー（Python版）仕様書
================================

■ 概要
  ブラウザで YouTube を再生し、別ウィンドウでリズムゲーム用のオーバーレイを表示する。
  本ゲーム本体は MP3 を再生しない。PC が取り込む再生音声（ループバック等）から
  音量・リズム（オンセット）を解析し、ノーツを自動生成する。
  ウィンドウ・ステージ・ノーツは半透明とし、背後の YouTube 映像を妨げにくくする。

  当面の対象動画例:
  【LIVE映像】結束バンド「あのバンド」LIVE at STARRY / 「ぼっち・ざ・ろっく！」劇中曲
  https://www.youtube.com/watch?v=5tc14WHUoMw&list=RD5tc14WHUoMw&start_radio=1
  同フォルダの *.url（Internet ショートカット）があればその URL をスタート時に開く。

■ 画面・入力
  ・解像度: 800×600
  ・レーン数: 4
  ・キー: D, F, J, K（上からレーン 0〜3）
  ・ESC: 終了
  ・ノーツ表示: 3D 透視投影（奥の z が大きく、手前の判定面で z が小さい）。
    投影式は (lane_x / z)*FOV + 中心、縦は (FLOOR_Y / z) + 消点オフセット。
  ・判定ライン: 3D 空間の手前面（z = Z_END 付近を画面に投影した位置）
  ・半透明: レーン床・ガイド線・ノーツ・（Windows で可能なら）ウィンドウ全体の透過

■ ゲームフロー
  1. タイトル画面で「スタート」（クリック / Space / Enter）
  2. 同一タイミングで既定ブラウザが YouTube を開く
  3. 音声入力ストリームの解析を開始（sounddevice）
  4. オンセット検出に応じてノーツを生成（レーンは乱数）
  5. ノーツが奥から手前へ近づく
  6. キー入力でタイミング判定・スコア・コンボ更新
  7. 解析メトリクスを画面右上に表示し、一定間隔で JSON に書き出し

■ 判定
  ・Perfect: ±0.05 秒
  ・Great: ±0.1 秒
  ・Good: ±0.2 秒
  ・Miss: 上記以外、または判定窓通過後の未処理

■ スコア
  ・Perfect: 100 点
  ・Great: 70 点
  ・Good: 50 点
  ・Miss: 0 点

■ タイム基準
  ・スタート押下を 0 秒とするセッション時間（time.perf_counter 系）
  ・任意で sync.json の time_offset_sec で全体オフセット

■ リアルタイム解析出力（JSON）
  ・ファイル名: live_audio_reactive.json（実行フォルダ）
  ・内容の例: RMS、平滑 RMS、推定 BPM、最後のオンセット強度、生成ノーツイベント履歴
  ・ゲーム本体のプレイ譜面はこのストリーム由来（chart.json は必須ではない）

■ オフライン譜面生成（別ツール）
  ・音源ファイル（例: music.mp3）から chart.json を生成するスクリプトあり
  ・形式: [ {"time": 秒, "lane": 0〜3}, ... ]
  ・generate_chart_from_audio.py / regenerate_chart.py
  ・既定配置モード bpm_volume（BPM グリッド + 区間内オンセット最大）、ほか onset / interval

■ 使用技術（目安）
  ・Python 3.10 以降推奨
  ・pygame（描画・入力）
  ・sounddevice + NumPy（リアルタイム音声入力・解析）
  ・librosa + NumPy（オフライン譜面生成）
  ・yt-dlp + ffmpeg（mp3download.py での音源取得）
