YouTube音ゲー（Python版）仕様書
================================

■ 概要
  ブラウザで YouTube を再生し、別ウィンドウの pygame で譜面を表示する。
  ノーツはリアルタイム音声解析ではなく、事前に MP3 から生成した譜面 JSON を使う。
  ゲーム起動時は URL から YouTube ID を抽出し、song_db.json と charts/*.json を参照して再生する。

■ フォルダ構成
  project/
    youtubeotoge.py
    generate_chart_from_mp3.py
    song_db.json
    charts/
      <youtube_id>.json
    mp3/
      01 - xxxx.mp3 〜 99 - xxxx.mp3（推奨命名）

■ 曲データベース（song_db.json）
  形式:
  {
    "songs": [
      {
        "youtube_id": "string",
        "title": "string",
        "youtube_title": "string",
        "duration_sec": float,
        "mp3": "filename.mp3",
        "offset": float
      }
    ]
  }

  ・youtube_id: YouTube URL から抽出（watch?v= / youtu.be 対応）
  ・duration_sec: MP3 解析結果（既存登録値を YouTube 側長さとして比較に利用）
  ・offset: 譜面時刻に加算する補正秒（note_time = chart_time + offset）

■ 譜面生成（generate_chart_from_mp3.py）
  入力:
  ・MP3（librosa で mono 読込）
  ・YouTube URL（または番号で自動紐づけ）

  処理:
  ・onset_strength / onset_detect でノーツ時刻候補を抽出
  ・テンポ推定（BPM）
  ・各オンセット時点のスペクトル重心から lane 0〜3 を決定
    - 0〜250 Hz      → lane 0
    - 250〜1500 Hz   → lane 1
    - 1500〜5000 Hz  → lane 2
    - 5000 Hz〜      → lane 3
  ・charts/<youtube_id>.json を保存
  ・song_db.json を upsert（同 youtube_id は更新）

■ 自動紐づけ
  ・mp3/ に 01-〜99- で始まる MP3 がある場合、引数なし実行で自動紐づけを開始
  ・youtube_url.txt の同じ番号行と MP3 を対応付ける
  ・未連携（song_db に youtube_id が未登録）番号を優先

■ 長さ差チェックと確認
  ・既存 song_db の同 youtube_id の duration_sec を YouTube 側長さとして扱う
  ・|mp3_duration - youtube_duration| > 2.0 秒 で確認表示
    「曲の長さが違いますが、jsonによりnoteを作成しますか？」

■ offset 自動判定
  ・曲長差がある場合、BPM/拍解析 + 先頭無音検出で offset を自動推定
  ・10秒を超えたらキャンセル
    「長くかかるので、自分で調べてね」
  ・手動指定: --offset / 手動入力モード（--manual-offset）

■ ゲーム側（youtubeotoge.py）
  ・タイトルで URL 選択
  ・extract_youtube_id(url) で ID 抽出
  ・song_db.json から曲情報取得
  ・charts/<youtube_id>.json をロード
  ・note_time = chart_time + offset で同期
  ・既存の pygame 描画 / キー入力 / 判定ロジックは維持
  ・リアルタイム音声入力ノーツ生成は無効化

■ 判定とスコア
  ・Perfect: ±0.05 秒（100点）
  ・Great: ±0.10 秒（70点）
  ・Good: ±0.20 秒（50点）
  ・Miss: 0点

■ 使用ライブラリ
  ・Python 3.10 以降推奨
  ・pygame
  ・numpy
  ・librosa
