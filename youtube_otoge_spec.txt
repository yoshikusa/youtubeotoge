YouTube音ゲー（Python版）仕様書
================================

■ 概要
  ブラウザで YouTube を再生し、別ウィンドウでリズムゲーム用のオーバーレイを表示する。
  音源は PC が取り込む再生音声（ループバック・ステレオミックス等）のみ。ローカル MP3 の
  再生やファイル譜面（chart.json）の読込は行わない。
  音量・オンセットからノーツ時刻を推定し、ループバック波形の周波数帯でレーン 0〜3 を割当。
  ウィンドウ・ステージ・ノーツは半透明とし、背後の YouTube 映像を妨げにくくする。

  再生 URL の読み込み優先順:
  1. youtube_urls.txt … 最大 20 行の http URL（タイトルで 1〜9・0=10、矢印・クリックで全件）
  2. youtube_url.txt … 単一 URL（候補が 1 件だけになる）
  3. *.url（Internet ショートカット）を順に最大 20 件まで
  4. コード内の既定 URL
  雛形: youtube_urls.example.txt（単一 URL は同ファイル冒頭の説明のとおり youtube_url.txt）
  ・タイトル画面で URL を選ぶと YouTube oEmbed でタイトル・サムネを取得し表示（要ネットワーク）
  ・起動画面は ↑↓ でリスト選択（1〜0 直接指定・クリックも可）。UI は TITLE_SCREEN_UI_SCALE で一括縮小

■ レーン割当（ループバック各ブロックの rFFT・帯域エネルギー最大）
  ・0〜250 Hz      → レーン 0（キック・青系）
  ・250〜1500 Hz   → レーン 1（スネア・黄）
  ・1500〜5000 Hz  → レーン 2（スネア・オレンジ寄り）
  ・5000 Hz〜      → レーン 3（ハイハット・赤系）
  実装: frequency_lanes.py（youtube_audio_capture が参照）

■ ノーツ生成（解析側）
  ・オンセット: 従来の立ち上がり検出。RMS が高いほど最小間隔を短くしサビで増える
  ・帯域: 正規化スペクトルが閾値を超えたレーンにノーツ（左上 4 本のバーと対応）
  ・RMS 追加: RMS > 約 0.02 で別系統のパルス間隔を詰めて追加ノーツ
  ・1 ブロックあたりの生成数に上限（静か時少／RMS 高いとき多め）

■ 画面・入力
  ・解像度: 800×600 基準×DISPLAY_SCALE
  ・レーン数: 4
  ・キー: D, F, J, K（上からレーン 0〜3）
  ・ESC: 終了
  ・ノーツ表示: 3D 透視投影（奥の z が大きく、手前の判定面で z が小さい）。
    投影式は (lane_x / z)*FOV + 中心、縦は (FLOOR_Y / z) + 消点オフセット。
  ・判定ライン: 3D 空間の手前面（z = Z_END 付近を画面に投影した位置）
  ・半透明: レーン床・ガイド線・ノーツ・（Windows で可能なら）ウィンドウ全体の透過

■ ゲームフロー
  1. タイトル画面で「スタート」（クリック / Space / Enter）
  2. 同一タイミングで既定ブラウザが YouTube を開く
  3. 音声入力ストリームの解析を開始（sounddevice・ループバック推奨）
  4. オンセット検出に応じてノーツ生成（レーンは上記周波数帯）
  5. ノーツが奥から手前へ近づく
  6. キー入力でタイミング判定・スコア・コンボ更新
  7. 解析メトリクスを画面右上に表示し、一定間隔で JSON に書き出し

■ 判定
  ・Perfect: ±0.05 秒
  ・Great: ±0.1 秒
  ・Good: ±0.2 秒
  ・Miss: 上記以外、または判定窓通過後の未処理

■ スコア
  ・Perfect: 100 点
  ・Great: 70 点
  ・Good: 50 点
  ・Miss: 0 点

■ タイム基準
  ・スタート押下を 0 秒とするセッション時間（time.perf_counter 系）
  ・任意で sync.json の time_offset_sec で全体オフセット

■ 表示設定（任意）
  ・ファイル名: display_settings.json（実行フォルダ＝スクリプトと同じ場所）
  ・無い場合: ゲーム中の右上メトリクスは小フォント既定（約 18×DISPLAY_SCALE px）
  ・hud_right_font_px: 右上テキストのフォントサイズ（整数、6〜120 程度を推奨）
  ・hud_right_line_gap_px: 行間（省略時はフォントの get_linesize() に合わせる）
  ・雛形: display_settings.example.json をコピーして編集

■ 音声入力デバイス（任意・番号指定）
  ・ループバック / Stereo Mix が名前判定で取れない場合、デバイス番号で固定できる
  ・python list_audio_devices.py … Index とデバイス名の一覧を表示
  ・audio_settings.json … {"input_device_index": 番号}（雛形: audio_settings.example.json）
  ・指定がある場合は自動ループバック検出より優先。番号が無効ならストリームは開始しない

■ リアルタイム解析出力（JSON）
  ・ファイル名: live_audio_reactive.json（実行フォルダ）
  ・内容の例: RMS、平滑 RMS、推定 BPM、最後のオンセット強度、生成ノーツイベント履歴

■ 使用技術（目安）
  ・Python 3.10 以降推奨
  ・pygame（描画・入力）
  ・sounddevice + NumPy（リアルタイム音声入力・解析）
